9. Producer Tool Manual (Complete Step-by-Step Guide)

The Gram Producer Tool is the internal interface used to create, edit, and manage Grams. Producers use it to upload artwork, configure metadata, assign perks, generate NFC URLs and QR codes, and save everything to the backend.

9.1 Accessing the Producer Tool

In production, the Producer Tool is served by the backend at: 

https://YOUR-BACKEND-URL/producer

When running locally during development: 

http://localhost:3000/producer

There is no authentication built into this tool yet, so you should keep it private by:

  • Restricting access by IP (only office / home network).
  • Placing it behind a password-protected reverse proxy (future improvement).
  • Only exposing it on internal networks or staging environments.

9.2 Layout & Sections

The Producer Tool UI is logically split into the following sections:

  • Images ‚Äì upload and manage artwork files.
  • Load Existing Gram ‚Äì load an existing Gram by ID or slug for editing.
  • Gram Details ‚Äì set ID, title, description, image URL, NFC tag ID, effects, slug.
  • Perks ‚Äì define discounts, free items, cooldowns and partner businesses.
  • Output & NFC ‚Äì JSON payload preview, backend save button, NFC URL.
  • QR Code ‚Äì generated from the NFC URL for printing or packaging.

9.3 Step 1 ‚Äî Upload Image(s)

The first step in creating a Gram is uploading the artwork image. The Producer Tool sends these files to Shopify Files through the backend.

  1. Click ‚ÄúChoose Files‚Äù and select one or more PNG/JPG files.
  2. Click ‚ÄúUpload to Shopify‚Äù.
  3. Wait until each file appears in the ‚ÄúUploaded images‚Äù list with a preview thumbnail.

Once uploaded, the backend:

  • Creates a staged upload in Shopify via GraphQL.
  • Uploads the binary file to Shopify‚Äôs S3 bucket.
  • Calls fileCreate to register it as a Shopify File.
  • Retrieves the final CDN URL and returns it to the Producer Tool.

The returned URL (e.g., https://cdn.shopify.com/s/files/.../GRAM_1_1.jpg?v=123456789) will be used as the image_url field for the Gram.

Troubleshooting Uploads

  • No images appear after uploading: Check the backend logs for Shopify errors.
  • Access denied / auth error: Verify SHOPIFY_ADMIN_TOKEN in Render env vars.
  • File too large: Compress image and try again.

9.4 Step 2 ‚Äî Load Existing Gram or Create a New One

Before filling details, decide if you are editing an existing Gram or creating a new one.

Load Existing Gram

You can load an existing Gram by:

  • Entering its ID (e.g., G006), or
  • Entering its slug (e.g., dreaming-siggy).

The tool will fetch the Gram from the backend and populate all fields and perks.

Create New Gram

For a new Gram:

  • Use the ‚ÄúGet Next ID‚Äù action (or similar) which calls /api/producer/next-id.
  • Alternatively, manually set a new ID following the existing format (G001, G002, G003...).

IDs should be unique and sequential for easier management. Avoid reusing or renaming IDs after Grams have been sold or assigned to NFC tags.

9.5 Step 3 ‚Äî Fill Gram Details

Now fill in all Gram properties in the ‚ÄúGram Details‚Äù section:

  • ID ‚Äì unique identifier (e.g., G006).
  • Title ‚Äì display name (e.g., ‚ÄúDreaming Siggy‚Äù).
  • Image URL ‚Äì select from the uploaded images or paste the Shopify File URL.
  • Description ‚Äì short description shown on the Gram page.
  • Slug ‚Äì URL-friendly version of the title (autogenerated, can be edited).
  • NFC Tag ID ‚Äì identifier printed or associated with the physical NFC tag (e.g., TAG-G006).
  • Effects ‚Äì optional frame/glow styles for the frontend to use.

The slug and NFC tag ID are particularly important. The slug powers shareable URLs and theme routing, and the NFC tag ID is what your backend uses to resolve a Gram from a physical tag.

9.6 Step 4 ‚Äî Configure Perks

Perks allow a Gram to act as a local rewards or benefits card. Each perk defines a relationship with a business and what benefit the owner gets.

Each perk includes fields like:

  • Business ID ‚Äì internal ID (e.g., AVOMARIA).
  • Business Name ‚Äì display name (e.g., ‚ÄúAvoMaria Caf√©‚Äù).
  • Type ‚Äì discount or free_item.
  • Discount % ‚Äì when type is discount (e.g., 10).
  • Item ‚Äì when type is free_item (e.g., ‚Äúespresso‚Äù).
  • Cooldown (seconds) ‚Äì minimum time between perk uses.

Example perks JSON as stored in Supabase:

{
  "perks": [
    {
      "business_id": "AVOMARIA",
      "business_name": "AvoMaria Caf√©",
      "type": "discount",
      "discount": 10,
      "cooldown": 86400
    },
    {
      "business_id": "BARDAESQUINA",
      "business_name": "Bar Da Esquina",
      "type": "free_item",
      "item": "beer",
      "cooldown": 86400
    }
  ]
}

You can add multiple perks for a single Gram so that one collectible provides benefits at different locations.

Perks Best Practices

  • Use stable, short business IDs to avoid breaking integrations.
  • Be explicit whether a perk is a discount or a free item.
  • Set realistic cooldowns (e.g., 24 hours = 86400 seconds).

9.7 Step 5 ‚Äî Review JSON Payload

Before saving, the Producer Tool shows a JSON preview of the Gram object that will be sent to the backend. 

{
  "id": "G006",
  "slug": "dreaming-siggy",
  "nfc_tag_id": "TAG-G006",
  "title": "Dreaming Siggy",
  "description": "Siggy sleeping under a tree dreaming of bones.",
  "image_url": "https://cdn.shopify.com/s/files/.../dreaming-siggy.jpg?v=123456",
  "effects": { "frame": "none", "glow": false },
  "owner_id": null,
  "perks": [ ... ]
}

Double-check:

  • ID is correct and not reused.
  • Slug matches the intended URL.
  • nfc_tag_id matches the physical NFC tag that will be programmed.
  • Perks array is correctly formatted JSON.

9.8 Step 6 ‚Äî Save Gram to Backend

When everything looks correct, click ‚ÄúSave to Backend‚Äù. The tool sends a POST request to /api/producer/grams.

The backend will:

  • Validate required fields (ID, slug, title, image_url).
  • Insert a new row if the ID doesn‚Äôt exist.
  • Update the existing row if the ID already exists.
  • Return a success or error response.

Common Save Errors

  • ‚ÄúMissing required Gram fields‚Äù ‚Äì Fill in ID, title, slug, and image URL.
  • Duplicate ID ‚Äì Use /next-id or choose a different ID.
  • Invalid JSON ‚Äì Fix formatting of perks or effects JSON.

9.9 Step 7 ‚Äî Generate NFC URL & QR Code

Once the Gram is saved, the Producer Tool generates the NFC URL based on the configured nfc_tag_id, for example: 

https://www.soldacstudio.com/gram?tag=TAG-G006

This URL is what you will write into the NFC tag using an NFC writer app, and what is encoded into the QR code.

The QR code is generated visually in the Producer Tool and can be:

  • Printed on packaging.
  • Used as a fallback when NFC is not available.
  • Shared digitally with testers or partners.

9.10 Editing an Existing Gram

To update an existing Gram (for example, to add new perks or improve the description):

  1. Load the Gram by ID or slug.
  2. Modify the desired fields (title, description, perks, effects, image).
  3. Review the JSON preview.
  4. Click ‚ÄúSave to Backend‚Äù again.

Fields you can safely edit anytime:

  • Title
  • Description
  • Image URL (if artwork changes)
  • Perks
  • Effects

Fields you should avoid changing after launch:

  • ID ‚Äì referenced in internal systems and backups.
  • Slug ‚Äì changing will break existing links.
  • NFC Tag ID ‚Äì changing requires reprogramming physical tags.
  • Owner ID ‚Äì changing ownership should be rare and manual via Supabase with caution.
